# 倉庫管理システム (Warehouse Management System)

## 概要

このシステムは、倉庫における入庫管理業務を支援するVB6ネイティブアプリケーションです。
入庫予定の管理、入庫実績の記録、ロケーション（棚）の管理を一元的に行うことができます。

## 機能概要

### 1. 入庫予定管理
入庫予定の登録・編集・削除を行う機能です。

**主な機能：**
- 入庫予定の一覧表示
- 新規入庫予定の登録
- 既存の入庫予定の編集
- 入庫予定の削除

**管理データ項目：**
- 発注番号：発注を識別する番号
- 商品コード：商品を識別するコード
- 数量：入庫予定数量
- 納期：入庫予定日

### 2. 入庫実績管理
実際の入庫作業の記録を管理する機能です。

**主な機能：**
- 入庫実績の一覧表示
- 新規入庫実績の登録（バーコードスキャン対応）
- 既存の入庫実績の編集
- 入庫実績の削除

**管理データ項目：**
- 発注番号：対応する発注番号
- 商品コード：入庫した商品のコード
- 数量：実際の入庫数量
- バーコードの種類：使用したバーコード形式
- 検品時エラー処理：検品時に発生したエラーとその対処
- 棚番号：割り当てた保管場所
- 入庫日：実際に入庫した日付

### 3. ロケーション管理
倉庫内の保管場所（棚）を管理する機能です。

**主な機能：**
- ロケーション（棚）の一覧表示
- 新規ロケーションの登録
- 既存ロケーションの編集
- ロケーションの削除

**管理データ項目：**
- 棚番号：棚を識別する番号
- 棚属性：棚の特性（常温、冷蔵、冷凍など）
- 割り当てルール：在庫の割り当て方法（先入先出、後入先出など）

## システム構成

### アーキテクチャ
- **開発言語：** Visual Basic 6.0
- **データベース：** インメモリデータベース（Collection型を使用）
- **UI：** VB6フォーム


## ビルド方法

### 必要な環境
- Windows OS（Windows 7以降推奨）
- Visual Basic 6.0（VB6）開発環境

### ビルド手順

1. **プロジェクトを開く**
   ```
   WarehouseSystem.vbp をVB6で開きます
   ```

2. **依存関係の確認**
   - プロジェクト → 参照設定から以下が参照されていることを確認：
     - Microsoft Windows Common Controls 6.0 (SP6) (MSCOMCTL.OCX)

3. **コンパイル**
   - メニューから「ファイル」→「WarehouseSystem.exeの作成」を選択
   - 保存場所を指定してコンパイル

4. **実行ファイルの配置**
   - コンパイルされた WarehouseSystem.exe を任意の場所に配置

### 実行方法

**開発環境での実行：**
- VB6でプロジェクトを開き、F5キーで実行

**コンパイル後の実行：**
- WarehouseSystem.exe をダブルクリック

## 使用方法

### 起動から操作までの流れ

1. **アプリケーションの起動**
   - WarehouseSystem.exe を実行すると、メインメニューが表示されます

2. **メインメニュー**
   - 「入庫予定」ボタン：入庫予定管理画面を開く
   - 「入庫実績」ボタン：入庫実績管理画面を開く
   - 「ロケーション管理」ボタン：ロケーション管理画面を開く

3. **各画面での操作**
   - **一覧表示：** 画面下部のリストに登録済みデータが表示されます
   - **新規登録：** 上部の入力フィールドにデータを入力し、「追加」ボタンをクリック
   - **編集：** リストから項目を選択し、入力フィールドの内容を変更後、「編集」ボタンをクリック
   - **削除：** リストから項目を選択し、「削除」ボタンをクリック
   - **閉じる：** 「閉じる」ボタンで画面を閉じます

### サンプルデータ

アプリケーション起動時に以下のサンプルデータが自動的に登録されます：

**入庫予定：**
- PO-001: PROD-001, 数量100, 納期3日後
- PO-002: PROD-002, 数量50, 納期5日後

**ロケーション：**
- A-01: 常温、先入先出
- A-02: 常温、先入先出
- B-01: 冷蔵、後入先出

## データの永続化について

現在のバージョンでは、データはメモリ上にのみ保存されます。
アプリケーションを終了すると、追加・編集したデータは失われ、次回起動時には初期サンプルデータの状態に戻ります。

データを永続化する場合は、以下のいずれかの方法を検討してください：
- ファイル（テキスト、CSV、XMLなど）への保存
- データベース（Access、SQL Serverなど）への接続
- レジストリへの保存

## セキュリティについて

このアプリケーションはサンプル実装のため、以下のセキュリティ機能は実装されていません：
- ユーザー認証・認可
- データの暗号化
- 操作ログの記録
- アクセス権限の管理

本番環境で使用する場合は、適切なセキュリティ対策を実装してください。

## トラブルシューティング

### アプリケーションが起動しない場合
- VB6ランタイムがインストールされているか確認
- MSCOMCTL.OCX が登録されているか確認

### ListView（一覧）が表示されない場合
- プロジェクトの参照設定でMicrosoft Windows Common Controls 6.0が参照されているか確認

### 日本語が文字化けする場合
- VB6のフォーム設計で適切な日本語フォント（MSゴシック、MS明朝など）が設定されているか確認
